Task #PointsTask Description
1 10 In the "rowColorSwitcher-XX.xlsm" workbook, you will find a module named "Solution."  In it, there is a subprocedure named "highlightRow."  Modify that sub procedure so that when it is executed, the entire row of the worksheet with the active cell is highlighted—that is, the background color of all the cells on the row is changed.  Your solution should highlight the whole row, even blank cells on the row.  Use the "colorindex" property of the "Interior" object to effect the change.  The first time the "highlightRow" sub procedure is called on a row, it should set the background color of the whole row to have a colorIndex of one of the following numbers: 3, 4, 6, 7, 8, -4142.  The value of -4142 is a special value that indicates no background color at all.  Regardless of the background color of the active cell, when the "highlightRow" sub procedure is called, it should change the background colorIndex of the entire row to one of the six valid numbers (3, 4, 6, 7, 8, -4142).

So, if a user clicks the button in the ribbon that calls the "highlightRow" sub procedure six times without changing the active cell, that row's background color should have shown all six different colors once.  If the user clicks again, it should change the color back to one of the six it has already shown.

Some students are tempted to use a loop in the creation of their solutions; however, each time the user clicks the "Highlight Row" button, only one color should be shown, so it is unlikely that a loop will help.
